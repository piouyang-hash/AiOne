## 📖 项目概述
SpringAI-Chat-Backend 是一款**完整自洽、开箱即用**的移动端AI聊天软件专属后端系统，专为移动端AI交互场景深度设计。项目采用最新的企业级技术栈构建，基于 **JDK21 + SpringBoot 4.0.6 + SpringAI 2.0.0-M7** 实现AI能力集成，使用 MyBatis-Plus 实现高效数据持久化，以MySQL作为唯一数据库支撑所有业务与AI核心功能。

系统实现了从**用户注册登录、AI智能对话、RAG检索增强、自定义智能体、积分计费**到**智能上下文管理**的全闭环功能，所有模块高度解耦、逻辑自洽，无需依赖第三方中间件即可独立运行，可直接对接移动端APP、小程序等前端产品，快速搭建具备商业化能力的AI聊天平台。

区别于通用AI后端项目，本项目聚焦**移动端交互体验**，针对前端断开连接、消息打断、流式输出、上下文压缩等实际场景做了深度优化，是一款面向生产环境的专业化AI聊天后端解决方案。

## 🎯 项目定位
- 面向移动端的**专属AI聊天后台**，适配APP/小程序等前端形态
- 零PG数据库依赖，纯MySQL实现RAG向量检索，降低部署与维护成本
- 具备完整用户体系、支付计费体系的**商业化AI后端**
- 高解耦、可扩展的AI会话引擎，支持自定义交互逻辑与上下文策略

## ✨ 核心功能模块（详细版）
### 1. 用户认证与登录模块（生产级标准）
本模块实现了移动端主流的安全登录体系，完全满足生产环境的安全与体验要求：
- 基础功能：新用户注册、账号密码登录、验证码校验、用户信息管理
- 核心能力：**双Token认证机制**（短时效Access Token + 长时效Refresh Token）
- 体验优化：**无感自动登录**，Token过期自动刷新，前端无感知切换
- 安全保障：Token签名加密、会话隔离、非法请求拦截
- 数据管理：用户状态持久化、登录日志记录

### 2. AI会话与消息管理模块（核心解耦引擎）
项目最核心的AI交互模块，实现了与前端、消息发送层的完全解耦：
- 基础功能：AI会话创建/删除/查询、聊天消息增删改查、历史消息回溯
- **解耦核心特性**：
  1. 前端断开网络连接 → AI后台持续生成回复，不会中断
  2. 前端手动打断AI回复 → 后台支持配置**继续回复句子数量**，模拟AI「意犹未尽」的拟人化效果
- 消息流式输出：自研**聊天消息切分函数**，实现类似微信的AI逐句回复效果，提升阅读体验
- 消息持久化：所有AI对话内容实时入库，支持历史记录全量检索

### 3. RAG检索增强生成模块（纯MySQL原生实现）
摒弃传统PG向量数据库依赖，**100%基于MySQL实现轻量化RAG功能**，部署极简：
- 核心功能：文本向量化存储、向量相似度检索、知识库问答
- 支持两种工业级检索算法：
  1. **余弦相似度**：适用于语义方向匹配
  2. **欧氏距离**：适用于向量空间距离匹配
- 数据流程：文本解析 → 向量生成 → MySQL存储 → 检索召回 → 注入AI上下文
- 优势：无需额外部署向量数据库，单机即可运行，运维成本极低

### 4. 自定义智能体模块（轻量化早期智能体）
面向任务型AI交互设计，支持用户自定义专属AI助手：
- 基础能力：固定系统提示词，强制AI执行指定任务、输出固定格式数据
- 核心功能：**用户自主创建/编辑/删除自定义智能体**，个性化配置提示词
- 适用场景：代码生成、文案写作、数据格式化、专业问答等
- 🚧 开发中：智能体社区（支持智能体分享、收藏、排行榜）

### 5. 积分支付与Token计费模块（商业化闭环）
完整的商业化计费体系，支持AI服务按量收费：
- 基础功能：用户积分充值、积分余额查询、积分消耗记录
- 计费规则：所有AI聊天对话**按规则消耗积分**，无积分无法发起对话
- 底层设计：双表驱动计费逻辑
  1. Token类型表：定义不同AI模型/服务的积分单价
  2. 单轮聊天Token消耗表：统计实际对话Token消耗量
- 核心实现：**责任链模式**处理积分消耗逻辑，解耦计费规则，便于扩展修改

### 6. 智能上下文管理模块（多策略动态适配）
解决大模型Token限制与对话连贯性的核心模块，支持多种上下文管理策略：
- 全维度上下文支撑：结构化信息、AI人格设定、对话总结摘要、长期向量记忆、历史会话记录
- 核心算法：**双滑动窗口机制**
  1. 普通滑动窗口：固定长度截断历史消息，平衡Token消耗
  2. 总结型滑动窗口：每N轮对话自动调用小模型生成摘要，强化AI对长对话的理解能力
- 优化策略：**热缓存纯追加策略**
  - 适用场景：热缓存会话，仅拼接历史消息，不做任何上下文压缩
  - 限制说明：冷启动会话/Token超限场景不推荐使用，逻辑持续迭代完善中

### 7. 🚧 开发中模块
- 分数与角色状态机制：用户行为评分、AI角色状态动态切换（开发中）

## 🏗️ 核心架构设计
1. **分层架构**：控制层 → 服务层 → 持久层 → AI引擎层，职责清晰
2. **解耦设计**：AI会话核心与前端连接状态、消息发送层完全分离，高可用
3. **可扩展架构**：支持快速接入新的大模型、新的计费规则、新的上下文策略
4. **统一数据层**：MySQL承载所有数据，无第三方中间件依赖

## 🚀 快速部署指南
### 一、环境要求（严格匹配）
1. JDK 21
2. MySQL 8.0+
3. Maven 3.8+
4. 任意主流IDE（IDEA推荐）

### 二、配置步骤
1. 克隆项目到本地
2. 修改 `application.yml` 核心配置：
   - MySQL数据库连接地址、账号、密码
   - SpringAI大模型API Key、模型版本配置
   - Token过期时间、积分规则、上下文窗口参数
3. 执行项目提供的SQL初始化脚本，创建所有业务表、向量表、计费表
4. Maven依赖下载：`mvn clean install`

### 三、启动项目
1. 运行SpringBoot启动类
2. 服务默认端口：`8080`
3. 启动成功后，可通过接口文档/前端对接测试全功能
